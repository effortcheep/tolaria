# 图片加载优化

## 一、图片加载的完整流程

从磁盘/网络到屏幕显示一张图片，经历了以下步骤：

```
磁盘/网络 → NSData → UIImage(CPU解码) → CGImage → GPU渲染 → 屏幕显示
```

**关键性能瓶颈：**
1. **I/O 读取** — 磁盘/网络读取耗时
2. **CPU 解码** — 将压缩数据（JPEG/PNG/HEIF）解码为位图，这是最大的性能瓶颈
3. **内存占用** — 解码后的位图占用大量内存
4. **GPU 渲染** — 将位图上传到 GPU 并渲染

---

## 二、图片解码原理

### 2.1 什么是解码

图片文件（JPEG/PNG/HEIF）是**压缩格式**，显示时需要解码为**位图（Bitmap）**数据。

```
压缩图片文件（几百KB）→ 解码 → 位图数据（可能几MB甚至几十MB）
```

### 2.2 解码后的内存计算

```
内存 = 图片宽度 × 图片高度 × 每像素字节数
```

| 像素格式 | 每像素字节 | 说明 |
|---------|-----------|------|
| RGBA 8888 | 4 bytes | 最常见，每个通道 8 bit |
| RGB 565 | 2 bytes | 无 Alpha，16 位色 |
| RGBA 16 Float | 8 bytes | HDR 图片 |
| BGRA 8888 | 4 bytes | iOS 常用 |

**示例：** 一张 3000×2000 的 JPEG 图片：
- 文件大小：约 2MB
- 解码后位图：3000 × 2000 × 4 = **24MB**

### 2.3 解码时机

图片解码发生在 **CoreAnimation 合成阶段**。当 UIImageView 的 image 被设置后，在下一次 RunLoop 的 `BeforeWaiting` 阶段，CoreAnimation 会触发解码。

```
[CATransaction flush] → 触发解码 → 合成 → 渲染
```

## 三、六大优化方案

### 3.1 异步解码（强制解码）

**原理：** 在子线程预先解码图片，避免主线程卡顿。

**方案一：手动解码**
```objc
+ (UIImage *)decodedImageWithImage:(UIImage *)image {
    CGImageRef imageRef = image.CGImage;
    if (!imageRef) return image;
    
    size_t width = CGImageGetWidth(imageRef);
    size_t height = CGImageGetHeight(imageRef);
    
    CGColorSpaceRef colorSpace = CGColorSpaceCreateDeviceRGB();
    
    CGContextRef context = CGBitmapContextCreate(
        NULL, width, height, 8, width * 4, colorSpace,
        kCGImageAlphaPremultipliedFirst | kCGBitmapByteOrder32Little
    );
    
    if (!context) {
        CGColorSpaceRelease(colorSpace);
        return image;
    }
    
    CGContextDrawImage(context, CGRectMake(0, 0, width, height), imageRef);
    CGImageRef decodedImageRef = CGBitmapContextCreateImage(context);
    UIImage *decodedImage = [UIImage imageWithCGImage:decodedImageRef
                                                scale:image.scale
                                          orientation:image.imageOrientation];
    
    CGContextRelease(context);
    CGColorSpaceRelease(colorSpace);
    CGImageRelease(decodedImageRef);
    
    return decodedImage;
}
```

**方案二：利用 ImageIO 框架（推荐）**
```objc
+ (UIImage *)decodedImageWithData:(NSData *)data {
    CGImageSourceRef source = CGImageSourceCreateWithData((__bridge CFDataRef)data, NULL);
    if (!source) return nil;
    
    NSDictionary *options = @{
        (id)kCGImageSourceShouldCache: @YES,
        (id)kCGImageSourceCreateThumbnailWithTransform: @YES,
        (id)kCGImageSourceThumbnailMaxPixelSize: @(1024)
    };
    
    CGImageRef imageRef = CGImageSourceCreateThumbnailAtIndex(source, 0, (__bridge CFDictionaryRef)options);
    UIImage *image = [UIImage imageWithCGImage:imageRef];
    
    CGImageRelease(imageRef);
    CFRelease(source);
    
    return image;
}
```

**方案三：UIGraphicsImageRenderer（iOS 10+）**
```objc
+ (UIImage *)decodedImageWithImage:(UIImage *)image {
    UIGraphicsImageRenderer *renderer = [[UIGraphicsImageRenderer alloc] initWithSize:image.size];
    return [renderer imageWithActions:^(UIGraphicsImageRendererContext *ctx) {
        [image drawAtPoint:CGPointZero];
    }];
}
```

### 3.2 降采样（Downsampling）

**原理：** 根据显示尺寸解码合适大小的位图，而非原始尺寸。

```objc
+ (UIImage *)downsampleImageAtURL:(NSURL *)url toSize:(CGSize)size scale:(CGFloat)scale {
    CGImageSourceRef source = CGImageSourceCreateWithURL((__bridge CFURLRef)url, NULL);
    if (!source) return nil;
    
    CGFloat maxDimension = MAX(size.width, size.height) * scale;
    
    NSDictionary *options = @{
        (id)kCGImageSourceCreateThumbnailFromImageAlways: @YES,
        (id)kCGImageSourceShouldCacheImmediately: @YES,
        (id)kCGImageSourceCreateThumbnailWithTransform: @YES,
        (id)kCGImageSourceThumbnailMaxPixelSize: @(maxDimension)
    };
    
    CGImageRef imageRef = CGImageSourceCreateThumbnailAtIndex(source, 0, (__bridge CFDictionaryRef)options);
    UIImage *image = [UIImage imageWithCGImage:imageRef scale:scale orientation:UIImageOrientationUp];
    
    CGImageRelease(imageRef);
    CFRelease(source);
    
    return image;
}
```

**内存对比：**

| 方式 | 3000×2000 图片显示在 300×200 的 UIImageView | 内存占用 |
|------|------------------------------------------|---------|
| 直接加载 | 解码 3000×2000 | 24MB |
| 降采样 | 解码 300×200 × scale | ~2.4MB |

### 3.3 图片格式选择

| 格式 | 压缩方式 | 透明度 | 适用场景 |
|------|---------|--------|---------|
| JPEG | 有损压缩 | 不支持 | 照片、复杂图像 |
| PNG | 无损压缩 | 支持 | 图标、需要透明的图 |
| HEIF | 有损压缩 | 支持 | iOS 11+，同等质量体积减半 |
| WebP | 有损/无损 | 支持 | 网络图片，比 JPEG 小 25-34% |
| GIF | 无损压缩 | 支持 | 动图 |
| SVG | 矢量 | 支持 | 图标、简单图形 |

### 3.4 图片缓存策略

**三级缓存架构：**
```
内存缓存（NSCache）→ 磁盘缓存（文件系统）→ 网络请求
```

**内存缓存实现：**
```objc
@interface ImageCache ()
@property (nonatomic, strong) NSCache *memoryCache;
@end

@implementation ImageCache

- (instancetype)init {
    self = [super init];
    if (self) {
        _memoryCache = [[NSCache alloc] init];
        _memoryCache.totalCostLimit = 50 * 1024 * 1024; // 50MB
        _memoryCache.countLimit = 100;
        
        [[NSNotificationCenter defaultCenter] addObserver:self
                                                 selector:@selector(clearMemory)
                                                     name:UIApplicationDidReceiveMemoryWarningNotification
                                                   object:nil];
    }
    return self;
}

- (void)setImage:(UIImage *)image forKey:(NSString *)key {
    NSUInteger cost = image.size.width * image.size.height * 4 * image.scale * image.scale;
    [self.memoryCache setObject:image forKey:key cost:cost];
    
    dispatch_async(dispatch_get_global_queue(DISPATCH_QUEUE_PRIORITY_DEFAULT, 0), ^{
        NSData *data = UIImagePNGRepresentation(image);
        [data writeToFile:[self diskPathForKey:key] atomically:YES];
    });
}

- (UIImage *)imageForKey:(NSString *)key {
    UIImage *image = [self.memoryCache objectForKey:key];
    if (image) return image;
    
    NSData *data = [NSData dataWithContentsOfFile:[self diskPathForKey:key]];
    if (data) {
        image = [UIImage imageWithData:data];
        if (image) {
            NSUInteger cost = image.size.width * image.size.height * 4 * image.scale * image.scale;
            [self.memoryCache setObject:image forKey:key cost:cost];
        }
    }
    return image;
}

@end
```

**NSCache 特点：** 线程安全、自动 LRU 清理、内存警告时自动清理、可设置 countLimit 和 totalCostLimit。

### 3.5 预加载与懒加载

**UICollectionView 预加载：**
```objc
- (void)collectionView:(UICollectionView *)collectionView
    prefetchItemsAtIndexPaths:(NSArray<NSIndexPath *> *)indexPaths {
    for (NSIndexPath *indexPath in indexPaths) {
        NSURL *imageURL = self.imageURLs[indexPath.item];
        [[ImageDownloader shared] prefetchImageWithURL:imageURL];
    }
}

- (void)collectionView:(UICollectionView *)collectionView
    cancelPrefetchingForItemsAtIndexPaths:(NSArray<NSIndexPath *> *)indexPaths {
    for (NSIndexPath *indexPath in indexPaths) {
        NSURL *imageURL = self.imageURLs[indexPath.item];
        [[ImageDownloader shared] cancelPrefetchWithURL:imageURL];
    }
}
```

**RunLoop 空闲加载：**
```objc
- (void)setupRunLoopObserver {
    CFRunLoopObserverRef observer = CFRunLoopObserverCreateWithHandler(
        kCFAllocatorDefault, kCFRunLoopBeforeWaiting, YES, 0,
        ^(CFRunLoopObserverRef observer, CFRunLoopActivity activity) {
            [self loadNextPendingImage];
        });
    CFRunLoopAddObserver(CFRunLoopGetMain(), observer, kCFRunLoopCommonModes);
    CFRelease(observer);
}
```

### 3.6 图片裁剪与圆角优化

```objc
// ❌ 触发离屏渲染
imageView.layer.cornerRadius = 50;
imageView.layer.masksToBounds = YES;

// ✅ 使用 Core Graphics 裁剪
- (UIImage *)roundedImageWithSize:(CGSize)size radius:(CGFloat)radius {
    UIGraphicsImageRenderer *renderer = [[UIGraphicsImageRenderer alloc] initWithSize:size];
    return [renderer imageWithActions:^(UIGraphicsImageRendererContext *ctx) {
        CGRect rect = CGRectMake(0, 0, size.width, size.height);
        UIBezierPath *path = [UIBezierPath bezierPathWithRoundedRect:rect cornerRadius:radius];
        [path addClip];
        [self drawInRect:rect];
    }];
}

// ✅ 使用 CAShapeLayer
CAShapeLayer *maskLayer = [CAShapeLayer layer];
maskLayer.path = [UIBezierPath bezierPathWithRoundedRect:imageView.bounds cornerRadius:50].CGPath;
imageView.layer.mask = maskLayer;
```

## 四、SDWebImage 核心实现

### 4.1 加载流程

```
1. 检查内存缓存 → 命中直接显示
2. 检查磁盘缓存 → 命中则异步解码显示
3. 发起网络请求 → 下载完成 → 写入磁盘缓存 → 异步解码 → 写入内存缓存 → 主线程显示
```

### 4.2 核心组件

```
SDWebImageManager
├── SDImageCache          // 缓存管理
│   ├── 内存缓存 (NSCache)
│   └── 磁盘缓存 (文件系统)
├── SDWebImageDownloader  // 下载管理
│   ├── SDWebImageDownloaderOperation
│   └── NSURLSession
└── SDImageCoder          // 编解码
    ├── SDImageIOCoder    // 系统格式
    ├── SDImageGIFCoder   // GIF
    └── SDImageWebPCoder  // WebP
```

### 4.3 关键优化点

```objc
// 1. 渐进式加载
[imageView sd_setImageWithURL:url
             placeholderImage:nil
                      options:SDWebImageProgressiveLoad];

// 2. 图片变换（在缓存前执行）
SDImageTransformer *transformer = [SDImageTransformer transformerWithBlock:^UIImage *(UIImage *image, NSString *key) {
    return [image sd_roundedCornerImageWithRadius:50 corners:UIRectCornerAllCorners borderWidth:0 borderColor:nil];
}];

// 3. 限制并发下载数
SDWebImageDownloader *downloader = [SDWebImageDownloader sharedDownloader];
downloader.maxConcurrentDownloads = 6;
downloader.downloadTimeout = 30.0;
```

## 五、性能对比与量化

| 指标 | 优化前 | 优化后 | 提升 |
|------|-------|-------|------|
| 列表滚动 FPS | 45fps | 58fps | +29% |
| 单张大图加载内存 | 85MB | 12MB | -86% |
| 列表内存峰值 | 320MB | 150MB | -53% |
| 首屏加载时间 | 2.8s | 1.2s | -57% |

**内存占用对比（3000×2000 JPEG，文件 2MB）：**
- 直接加载：2MB + 24MB = 26MB
- 降采样到 300×200：2MB + 0.24MB = 2.24MB
- 降采样 + ImageIO 即时解码：仅 0.24MB

## 六、调试与检测

```objc
// 检测图片内存占用
- (void)logImageMemory:(UIImage *)image {
    CGImageRef cgImage = image.CGImage;
    size_t width = CGImageGetWidth(cgImage);
    size_t height = CGImageGetHeight(cgImage);
    size_t bytesPerRow = CGImageGetBytesPerRow(cgImage);
    NSLog(@"图片尺寸: %zu × %zu, 内存占用: %.2f MB", width, height, (height * bytesPerRow) / 1024.0 / 1024.0);
}
```

**Instruments 工具：** Time Profiler（解码耗时）、Allocations（内存分配）、Core Animation（离屏渲染）、Network（网络请求）

**常见问题排查：**

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 列表滚动卡顿 | 主线程解码大图 | 异步解码 + 降采样 |
| 内存暴涨 | 未使用缓存或缓存过大 | 设置合理的缓存限制 |
| 图片模糊 | 降采样比例过大 | 根据显示尺寸计算合适的采样率 |
| 闪白 | 异步解码未完成就显示 | 先显示缩略图/占位图 |
| OOM 崩溃 | 大图同时加载过多 | 限制并发数 + 降采样 |

## 七、面试回答模板

> **图片加载优化主要从以下几个层面入手：**
>
> **第一，解码优化。** 图片文件是压缩格式（JPEG/PNG），显示前需要解码为位图。一张 3000×2000 的 JPEG 解码后占用 24MB 内存。优化方案是异步解码，在子线程提前解码，避免阻塞主线程。
>
> **第二，降采样。** 根据实际显示尺寸解码，而不是解码原始尺寸。使用 ImageIO 的 `CGImageSourceCreateThumbnailAtIndex` 可以在解码阶段直接降采样，内存占用可以从 24MB 降到 2MB。
>
> **第三，缓存策略。** 采用内存+磁盘二级缓存，内存缓存用 NSCache（线程安全、自动清理），磁盘缓存用文件系统。设置合理的缓存上限，监听内存警告自动清理。
>
> **第四，格式选择。** iOS 11+ 支持 HEIF，同等质量体积减半。网络图片优先 WebP，比 JPEG 小 25-34%。
>
> **第五，列表优化。** UICollectionView 使用 prefetching 预加载，结合降采样和异步解码。圆角处理避免离屏渲染，用 Core Graphics 预裁剪代替 layer.cornerRadius。
>
> 实际项目中，我使用 SDWebImage 作为基础，配合 ImageIO 降采样和自定义缓存策略，列表 FPS 从 45 提升到 58，内存峰值降低 53%。
