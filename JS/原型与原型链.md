# 原型与原型链

## 什么是原型

JavaScript 中每个**对象**都有一个内部属性 `[[Prototype]]`（即隐式原型），它指向另一个对象——这个对象就是它的**原型**。

原型本身也是一个对象，也有自己的原型，这样层层向上，就形成了**原型链**。

原型的核心作用：**属性和方法的共享与继承**。

```javascript
const obj = { name: 'Tom' }
// obj.__proto__ 指向 Object.prototype
// Object.prototype.__proto__ 指向 null（原型链终点）
```

## 隐式原型 vs 显式原型

这是面试中经常被追问的区别：

| | 隐式原型 | 显式原型 |
|------|--------|--------|
| 属性名 | `__proto__` | `prototype` |
| 存在于 | **所有对象** | **只有函数** |
| 来源 | 创建对象时 JS 引擎自动设置 | 函数声明/定义时自动创建 |
| 指向 | 创建该对象的构造函数的 `prototype` | 该函数作为构造函数时，实例共享的原型对象 |
| 能否直接修改 | 可以但不推荐 | 可以，这是实现继承的常用手段 |

```javascript
function Person() {}

// 显式原型：Person 函数自带的 prototype 属性
Person.prototype  // { constructor: Person }

// 隐式原型：实例的 __proto__，指向构造函数的 prototype
const p = new Person()
p.__proto__ === Person.prototype  // true
```

### 为什么叫"隐式"和"显式"？

- **显式**（`prototype`）：函数对象上明确定义的属性，可以直接访问和修改
- **隐式**（`__proto__`）：JS 引擎在创建对象时**自动**设置的，早期没有标准方式访问，后来才通过 `__proto__` 暴露出来（现已被 ES2015 标准化为 `Object.getPrototypeOf()`）

```javascript
function Foo() {}
const f = new Foo()

// 显式：函数 Foo 身上直接有的属性
Foo.prototype  // { constructor: Foo }

// 隐式：实例 f 的 __proto__，不是 f 自己的属性，而是连接到原型的桥梁
f.__proto__ === Foo.prototype  // true

// 等价的标准化写法
Object.getPrototypeOf(f) === Foo.prototype  // true
```

### 一个经典易错点

```javascript
function Foo() {}
const f = new Foo()

// f 没有 prototype 属性（只有函数才有）
f.prototype  // undefined

// 但 f 有 __proto__（所有对象都有）
f.__proto__  // Foo.prototype

// 原型对象本身也有 __proto__
Foo.prototype.__proto__  // Object.prototype
```

**记住**：`prototype` 是"蓝图"，`__proto__` 是"查找的入口"。

## 核心三者关系

| 属性 | 存在于 | 指向 | 说明 |
|------|--------|------|------|
| `prototype` | **函数** | 原型对象 | 只有函数才有，构造函数的蓝图 |
| `__proto__` | **所有对象** | 创建该对象时所用构造函数的 prototype | 指向原型，是查找的入口 |
| `constructor` | **原型对象** | 构造函数本身 | 反向指回构造函数 |

```javascript
function Person(name) {
  this.name = name
}

Person.prototype.sayHi = function() {
  console.log('Hi, I am ' + this.name)
}

const p = new Person('Tom')

// 三者关系
p.__proto__ === Person.prototype        // true
Person.prototype.constructor === Person // true
p.constructor === Person                // true（沿原型链找到的）
```

## 原型链查找机制

当访问对象的一个属性时，查找顺序：

```
1. 先在对象自身找有没有这个属性
   └── 找到 → 返回

2. 自身没有 → 沿着 __proto__ 去构造函数的 prototype 上找
   └── 找到 → 返回

3. prototype 上也没有 → 沿着 __proto__ 去上一级 prototype 上找
   └── 找到 → 返回

4. ……一直往上找……

5. 到 Object.prototype 上找（toString、hasOwnProperty 都在这里）
   └── 找到 → 返回

6. Object.prototype 也没有 → 沿着 __proto__ 到 null
   └── null → 返回 undefined，找不到
```

这就是**原型链查找**，沿着 `__proto__` 一路往上找，直到 `null`。

```javascript
function Animal(name) {
  this.name = name
}
Animal.prototype.eat = function() { return 'eating' }

function Dog(name) {
  Animal.call(this, name)
}
Dog.prototype = Object.create(Animal.prototype)
Dog.prototype.constructor = Dog
Dog.prototype.bark = function() { return 'woof' }

const d = new Dog('Buddy')

// 查找过程
d.bark()   // 自身 → Dog.prototype → 找到
d.eat()    // 自身 → Dog.prototype → Animal.prototype → 找到
d.toString // 自身 → Dog.prototype → Animal.prototype → Object.prototype → 找到
d.xyz      // 自身 → Dog.prototype → Animal.prototype → Object.prototype → null → undefined
```

## 对象创建方式与原型关系

### 1. 字面量

```javascript
const obj = { a: 1 }
// obj.__proto__ === Object.prototype
// 等价于 const obj = new Object({ a: 1 })
```

### 2. 构造函数

```javascript
function Foo() {}
const f = new Foo()
// f.__proto__ === Foo.prototype
// Foo.prototype.__proto__ === Object.prototype
// Foo.__proto__ === Function.prototype
```

### 3. Object.create

```javascript
const parent = { greet() { return 'hello' } }
const child = Object.create(parent)
// child.__proto__ === parent
// 不经过任何构造函数的 prototype
```

### 4. class（ES6 语法糖）

```javascript
class Animal {
  constructor(name) { this.name = name }
  speak() { return this.name + ' speaks' }
}

class Dog extends Animal {
  bark() { return this.name + ' barks' }
}

const d = new Dog('Rex')
// 原型链: d → Dog.prototype → Animal.prototype → Object.prototype → null
```

## instanceof 原理

`instanceof` 沿着左操作数的原型链查找，看右操作数的 `prototype` 是否在链上。

```javascript
// 手动实现
function myInstanceof(obj, Constructor) {
  let proto = Object.getPrototypeOf(obj)
  while (proto !== null) {
    if (proto === Constructor.prototype) return true
    proto = Object.getPrototypeOf(proto)
  }
  return false
}

// 验证
myInstanceof([], Array)   // true
myInstanceof([], Object)  // true（Array.prototype → Object.prototype）
myInstanceof({}, Array)   // false
```

### instanceof 的局限

```javascript
// 不能检测基本类型
1 instanceof Number            // false
'hello' instanceof String      // false

// 跨 iframe 失败（Array 来自不同全局环境）
// 用 Array.isArray() 更可靠
```

## 原型链的终点

```javascript
Object.prototype.__proto__ === null  // true

// 验证
const obj = {}
let p = obj
while (p) {
  console.log(p)
  p = Object.getPrototypeOf(p)
}
// 输出: {} → Object.prototype → null
```

`null` 是原型链的终点，表示"没有更多原型了"。

## 常见面试问题

### 1. 什么是原型链？

对象通过 `__proto__` 串联起来的链式结构。访问属性时沿链向上查找，直到 `null`。

### 2. prototype 和 __proto__ 的区别？

- `prototype` 是**函数**才有的属性，指向构造函数的原型对象
- `__proto__` 是**所有对象**都有的，指向创建它的构造函数的 `prototype`

### 3. 如何判断一个属性是自身属性还是继承属性？

```javascript
const obj = { a: 1 }
obj.hasOwnProperty('a')     // true（自身）
obj.hasOwnProperty('toString') // false（继承自 Object.prototype）

// in 操作符会查整条链
'a' in obj          // true
'toString' in obj   // true
```

### 4. new 操作符内部做了什么？

```javascript
function myNew(Constructor, ...args) {
  const obj = Object.create(Constructor.prototype) // 1. 创建空对象，连接原型
  const result = Constructor.apply(obj, args)      // 2. 执行构造函数，绑定 this
  return result instanceof Object ? result : obj   // 3. 返回对象（构造函数有返回值时特殊处理）
}
```

### 5. 为什么 `typeof null === 'object'`？

这是 JS 的历史 Bug。`null` 的二进制表示全为 0，与对象的类型标签相同，`typeof` 无法区分。但这**不代表** `null` 在原型链上有位置——`null` 是原型链的终点。

### 6. `Object.create(null)` 创建的对象有什么特点？

```javascript
const naked = Object.create(null)
naked.__proto__          // undefined
naked instanceof Object  // false
naked.toString           // undefined（没有继承任何方法）

// 常用作纯净的字典，避免原型污染
```

### 7. 如何判断一个对象是否在另一个对象的原型链上？

```javascript
// isPrototypeOf
Animal.prototype.isPrototypeOf(d)  // true

// 或者手动沿链查找（上面的 myInstanceof）
```

## 总结

- 每个对象都有 `__proto__`，指向它的原型
- 函数额外有 `prototype`，指向实例共享的原型对象
- 原型链是对象属性查找的核心机制
- 终点是 `Object.prototype.__proto__ === null`
- `new`、`Object.create`、`class extends` 都在构建原型链
