# 闭包

## 什么是闭包

闭包（Closure）是指**函数能够记住并访问其词法作用域，即使函数在该作用域之外执行**。

简单说：闭包 = 函数 + 词法环境

```javascript
function outer() {
  const count = 0
  function inner() {
    console.log(count) // inner 访问了 outer 的变量
  }
  return inner
}

const fn = outer() // outer 执行完毕
fn() // 0 —— inner 仍然能访问 count
```

## 闭包的形成条件

1. **函数嵌套** — 内部函数引用外部函数的变量
2. **内部函数被返回或传递到外部** — 脱离原始作用域
3. **外部函数执行完毕** — 但内部函数仍持有引用

## 闭包的本质：词法作用域

JavaScript 采用**词法作用域**（静态作用域），函数的作用域在**定义时**确定，而非调用时。

```javascript
const x = 10

function foo() {
  console.log(x) // 定义在全局，所以访问全局的 x
}

function bar() {
  const x = 20
  foo() // 调用位置在 bar 内部，但 foo 的作用域仍是全局
}

bar() // 10，不是 20
```

闭包利用这个特性：内部函数在定义时"捕获"了外部变量的引用。

## 闭包的作用

### 1. 数据私有化

```javascript
function createCounter() {
  let count = 0 // 外部无法直接访问
  return {
    increment() { return ++count },
    decrement() { return --count },
    getCount() { return count }
  }
}

const counter = createCounter()
counter.increment()
counter.increment()
console.log(counter.getCount()) // 2
console.log(count) // ReferenceError: count is not defined
```

### 2. 函数工厂

```javascript
function multiply(factor) {
  return function(number) {
    return number * factor
  }
}

const double = multiply(2)
const triple = multiply(3)

console.log(double(5))  // 10
console.log(triple(5))  // 15
```

### 3. 柯里化

```javascript
function add(a) {
  return function(b) {
    return a + b
  }
}

const add5 = add(5)
console.log(add5(3))  // 8
console.log(add5(7))  // 12
```

## 经典问题：循环中的闭包

### 问题代码

```javascript
for (var i = 0; i < 3; i++) {
  setTimeout(() => {
    console.log(i)
  }, 100)
}
// 输出：3 3 3（不是 0 1 2）
```

**原因**：`var` 声明的 `i` 是函数级作用域，三个回调共享同一个 `i`，执行时循环已结束，`i` 为 3。

### 解决方案 1：使用 `let`

```javascript
for (let i = 0; i < 3; i++) {
  setTimeout(() => {
    console.log(i)
  }, 100)
}
// 输出：0 1 2
```

`let` 有块级作用域，每次迭代创建新的 `i`。

### 解决方案 2：IIFE 创建闭包

```javascript
for (var i = 0; i < 3; i++) {
  (function(j) {
    setTimeout(() => {
      console.log(j)
    }, 100)
  })(i)
}
// 输出：0 1 2
```

IIFE 立即执行，为每次迭代创建独立的作用域。

### 解决方案 3：`setTimeout` 的第三个参数

```javascript
for (var i = 0; i < 3; i++) {
  setTimeout((j) => {
    console.log(j)
  }, 100, i) // 第三个参数作为回调的参数
}
// 输出：0 1 2
```

## 闭包的内存管理

### 内存泄漏风险

闭包会保持对外部变量的引用，如果不释放，可能导致内存泄漏：

```javascript
function heavyData() {
  const data = new Array(1000000).fill('💀')
  return function() {
    console.log(data.length) // data 不会被回收
  }
}

const fn = heavyData() // data 常驻内存
// 不再需要时
fn = null // 释放引用，data 可被回收
```

### V8 的优化

现代引擎会分析闭包实际引用的变量，未使用的变量不会被保留：

```javascript
function outer() {
  const a = 1
  const b = 2 // 大对象
  return function inner() {
    return a // 只引用了 a，b 可被回收
  }
}
```

## 闭包的实际应用

### 1. 防抖（Debounce）

```javascript
function debounce(fn, delay) {
  let timer = null
  return function(...args) {
    clearTimeout(timer)
    timer = setTimeout(() => {
      fn.apply(this, args)
    }, delay)
  }
}

const handleSearch = debounce((keyword) => {
  console.log('Searching:', keyword)
}, 300)
```

### 2. 节流（Throttle）

```javascript
function throttle(fn, interval) {
  let lastTime = 0
  return function(...args) {
    const now = Date.now()
    if (now - lastTime >= interval) {
      lastTime = now
      fn.apply(this, args)
    }
  }
}
```

### 3. 模块模式

```javascript
const Module = (function() {
  let privateVar = 'secret'
  
  return {
    getSecret() {
      return privateVar
    },
    setSecret(value) {
      privateVar = value
    }
  }
})()

console.log(Module.getSecret()) // 'secret'
console.log(privateVar) // ReferenceError
```

## 常见面试问题

### 1. 什么是闭包？

**答**：闭包是函数与其词法环境的组合。内部函数可以访问外部函数的变量，即使外部函数已执行完毕。

### 2. 闭包的作用？

**答**：
- 数据私有化/封装
- 函数工厂和柯里化
- 维护状态在函数调用之间
- 模块模式

### 3. 闭包的缺点？

**答**：
- 内存泄漏风险（变量不被回收）
- 比普通函数占用更多内存
- 可能导致意外的副作用

### 4. 如何解决循环中的闭包问题？

**答**：
- 使用 `let` 替代 `var`
- 使用 IIFE 创建独立作用域
- 使用 `setTimeout` 的第三个参数

### 5. 闭包和作用域链的关系？

**答**：闭包利用词法作用域的特性，在定义时捕获外部变量的引用。作用域链是查找变量的机制，闭包让这个链在函数执行后仍然保持。

## 核心要点

- 闭包 = 函数 + 词法环境
- 闭包在**定义时**确定作用域，不是调用时
- 闭包让函数能"记住"并访问外部变量
- 循环中的闭包问题用 `let` 或 IIFE 解决
- 注意内存管理，不再需要时释放引用
