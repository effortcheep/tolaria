# 事件冒泡、捕获与事件委托

## 什么是事件流

当用户点击页面上某个元素时，这个点击事件不会只发生在该元素上，而是经历了三个阶段：

```
捕获阶段（Capture）   →  从 window 向下传播到目标
目标阶段（Target）     →  事件到达目标元素
冒泡阶段（Bubble）    →  从目标向上冒泡到 window
```

```
         window
           ↓  捕获
         document
           ↓
       <html>
           ↓
        <body>
           ↓
        <div>    ← 父元素
           ↓
         <p>     ← 目标（Target）
           ↑
        <div>    ← 父元素
           ↑  冒泡
        <body>
           ↑
       <html>
           ↑
         document
           ↑
         window
```

## 事件捕获（Capture）

事件从最外层的 `window` 开始，逐层向下传播，直到到达目标元素。

```javascript
element.addEventListener('click', handler, true)  // 第三个参数为 true
// 或
element.addEventListener('click', handler, { capture: true })
```

**特点：**
- 事件从外向内传播
- 父元素先于子元素触发
- 早期浏览器（IE8 及以下）不支持

## 事件冒泡（Bubble）

事件从目标元素开始，逐层向上传播，直到 `window`。

```javascript
element.addEventListener('click', handler)  // 默认就是冒泡
element.addEventListener('click', handler, false)  // 等价写法
```

**特点：**
- 事件从内向外传播
- 子元素先于父元素触发
- 绝大多数事件都会冒泡（`focus`、`blur`、`mouseenter`、`mouseleave` 不冒泡）

## 事件委托（Event Delegation）

利用事件冒泡机制，把子元素的事件处理逻辑委托给父元素统一管理。

```html
<ul id="list">
  <li>item 1</li>
  <li>item 2</li>
  <li>item 3</li>
</ul>
```

```javascript
// ❌ 错误做法：给每个 li 都绑定事件
document.querySelectorAll('li').forEach(li => {
  li.addEventListener('click', () => {
    console.log(li.textContent)
  })
})

// ✅ 正确做法：事件委托
document.getElementById('list').addEventListener('click', (e) => {
  if (e.target.tagName === 'LI') {
    console.log(e.target.textContent)
  }
})
```

### 事件委托的优势

| 场景 | 直接绑定 | 事件委托 |
|------|----------|----------|
| 内存消耗 | N 个子元素 = N 个监听器 | 1 个监听器 |
| 动态子元素 | 新增子元素需要单独绑定 | 自动生效 |
| 代码维护 | 分散在各处 | 集中管理 |
| 初始化性能 | 多次 DOM 操作 | 一次 DOM 操作 |

### 事件委托的注意事项

**1. `e.target` vs `e.currentTarget`**

```javascript
document.getElementById('list').addEventListener('click', (e) => {
  console.log(e.target)       // 实际点击的元素（可能是 li 内的 span）
  console.log(e.currentTarget) // 绑定事件的元素（始终是 #list）
})
```

**2. 处理多层嵌套**

```html
<ul id="list">
  <li>
    <span>item 1</span>
    <button>删除</button>
  </li>
</ul>
```

```javascript
document.getElementById('list').addEventListener('click', (e) => {
  // 用 closest 向上查找匹配的元素
  const li = e.target.closest('li')
  if (!li) return

  if (e.target.closest('button')) {
    console.log('点击了删除按钮')
  } else {
    console.log('点击了 li')
  }
})
```

**3. 不冒泡的事件无法委托**

以下事件不冒泡，不能使用事件委托：
- `focus` / `blur`（可用 `focusin` / `focusout` 替代）
- `mouseenter` / `mouseleave`（可用 `mouseover` / `mouseout` 替代）
- `load` / `unload` / `scroll`

## 阻止事件传播

### `e.stopPropagation()`

阻止事件继续传播（捕获或冒泡阶段都会生效）。

```javascript
child.addEventListener('click', (e) => {
  e.stopPropagation() // 阻止冒泡到父元素
  console.log('child')
})

parent.addEventListener('click', () => {
  console.log('parent') // 不会执行
})
```

### `e.stopImmediatePropagation()`

阻止事件传播，并且阻止当前元素上**同事件类型**的其他监听器执行。

```javascript
element.addEventListener('click', (e) => {
  e.stopImmediatePropagation()
  console.log('first') // ✅ 执行
})

element.addEventListener('click', () => {
  console.log('second') // ❌ 不执行
})
```

### `e.preventDefault()`

阻止浏览器的默认行为，**不阻止事件传播**。

```javascript
a.addEventListener('click', (e) => {
  e.preventDefault() // 阻止跳转，但事件继续冒泡
})

form.addEventListener('submit', (e) => {
  e.preventDefault() // 阻止表单默认提交
})
```

### 对比

| 方法 | 阻止传播 | 阻止默认行为 | 阻止同元素其他监听器 |
|------|----------|-------------|---------------------|
| `stopPropagation()` | ✅ | ❌ | ❌ |
| `stopImmediatePropagation()` | ✅ | ❌ | ✅ |
| `preventDefault()` | ❌ | ✅ | ❌ |

## 事件捕获 vs 冒泡的执行顺序

```html
<div id="parent">
  <button id="child">点击</button>
</div>
```

```javascript
const parent = document.getElementById('parent')
const child = document.getElementById('child')

parent.addEventListener('click', () => console.log('parent 捕获'), true)
parent.addEventListener('click', () => console.log('parent 冒泡'), false)
child.addEventListener('click', () => console.log('child 捕获'), true)
child.addEventListener('click', () => console.log('child 冒泡'), false)
```

输出：
```
parent 捕获
child 捕获
child 冒泡
parent 冒泡
```

**顺序规则：**
1. 捕获阶段：从外到内
2. 目标阶段：按代码注册顺序（先注册先执行）
3. 冒泡阶段：从内到外

## `event.target` 与事件流

```javascript
outer.addEventListener('click', (e) => {
  console.log(e.target)       // 始终是实际点击的元素
  console.log(e.currentTarget) // 当前处理事件的元素
  console.log(e.eventPhase)    // 1=捕获 2=目标 3=冒泡
}, true)
```

## 常见面试问题

### 1. 事件冒泡和事件捕获的区别？

| 捕获 | 冒泡 |
|------|------|
| 从 window 向目标传播 | 从目标向 window 传播 |
| 父元素先触发 | 子元素先触发 |
| `addEventListener` 第三个参数为 `true` | 第三个参数为 `false`（默认） |
| 先执行 | 后执行 |

### 2. 什么是事件委托？有什么好处？

利用**事件冒泡**机制，将子元素的事件处理委托给父元素。好处：减少内存消耗、动态元素自动生效、代码集中管理。

### 3. `e.target` 和 `e.currentTarget` 的区别？

- `e.target`：**实际触发事件的元素**（用户真正点击的那个）
- `e.currentTarget`：**绑定事件监听器的元素**（`this` 指向的元素）

### 4. `preventDefault()` 和 `stopPropagation()` 的区别？

- `preventDefault()`：阻止浏览器默认行为（如链接跳转、表单提交），事件继续传播
- `stopPropagation()`：阻止事件继续传播（捕获或冒泡），不影响默认行为

### 5. 哪些事件不冒泡？

`focus`、`blur`、`mouseenter`、`mouseleave`、`load`、`unload`、`scroll`。替代方案：`focusin`/`focusout`、`mouseover`/`mouseout`。

### 6. 如何让事件只在捕获阶段触发？

```javascript
element.addEventListener('click', handler, { capture: true, once: true })
// once: true 表示只触发一次
```

### 7. Vue 和 React 中的事件机制？

**Vue：**
- Vue 2：事件绑定在真实 DOM 上，遵循原生事件流
- Vue 3：事件绑定在组件根元素上，部分利用了事件委托

**React：**
- React 16：事件委托在 `document` 上
- React 17+：事件委托在**根节点**（`#root`）上
- React 使用合成事件（SyntheticEvent），不是原生事件对象

```javascript
// React 阻止冒泡用 e.stopPropagation()
// 阻止默认行为用 e.preventDefault()
// 注意：React 中 onClick 中调用 e.stopPropagation() 阻止的是 React 合成事件的冒泡
// 如果需要阻止原生事件冒泡，需要直接操作 DOM
```

### 8. 事件委托的局限性？

1. **不冒泡的事件无法委托**（focus、blur 等）
2. **层级过深时 `e.target` 判断复杂**（需要用 `closest`）
3. **某些场景下事件委托反而不好**：如 `mousemove` 频繁触发时，委托到父元素会增加判断开销
