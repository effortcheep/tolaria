# 虚拟 DOM 与 Diff 算法

## 一、什么是虚拟 DOM（Virtual DOM）

虚拟 DOM 是对真实 DOM 树的 **JavaScript 对象映射**。它用轻量的 JS 对象描述 DOM 结构，不直接操作真实 DOM。

```
// 真实 DOM 节点 → 虚拟 DOM 对象
<div class="box" id="app">Hello</div>
↓
{
  tag: 'div',
  props: { class: 'box', id: 'app' },
  children: ['Hello']
}
```

## 二、为什么需要虚拟 DOM

| 痛点 | 虚拟 DOM 的解决方案 |
|------|-------------------|
| 直接操作 DOM 代价高（重排重绘） | 批量计算差异，最小化真实 DOM 操作 |
| 手动追踪状态变化容易出错 | 框架自动 diff，声明式编程 |
| 跨平台困难 | 虚拟节点可渲染到不同平台（浏览器、Native、SSR） |

> **核心价值**：不是"比直接操作 DOM 快"，而是提供 **声明式 + 高效更新** 的开发体验。

## 三、Diff 算法的核心思想

传统树 diff 算法复杂度为 **O(n³)**，不可接受。React/Vue 通过三个策略降到 **O(n)**：

### 1. 同层比较（Same Level）
只比较同一层级的节点，不做跨层移动。

```
旧:        新:
 A          A
/ \        / \
B   C      B   D    ← 只比较同层，发现 C→D
```

### 2. 同类型比较（Same Type）
节点类型不同 → 直接销毁重建，不继续递归比较子树。

```
旧: <div>     新: <span>
      ↓ 类型不同 → 整棵子树替换
```

### 3. Key 标识（Unique Key）
通过 `key` 属性识别哪些节点是"同一个"，实现高效复用。

```jsx
// React
<li key={item.id}>{item.name}</li>
```

## 四、Vue 2 的 Diff — 双端比较

Vue 2 使用 **双端比较**（Double End）策略，用四个指针从两端向中间逼近：

```
旧列表: [A, B, C, D, E]
新列表: [A, D, B, C, F]

四步比较（每轮）:
1. 旧头 vs 新头  → 匹配则指针后移
2. 旧尾 vs 新尾  → 匹配则指针前移
3. 旧头 vs 新尾  → 匹配则旧头节点移到尾部
4. 旧尾 vs 新头  → 匹配则旧尾节点移到头部

都不匹配 → 用 key 在旧列表中查找，找到则移动，找不到则创建
```

**优势**：对常见的列表头部/尾部插入删除场景，效率极高。

## 五、Vue 3 的优化 — 最长递增子序列

Vue 3 在双端比较基础上引入了 **最长递增子序列（LIS）** 优化：

```
旧: [A, B, C, D, E]
新: [A, C, E, B, D]

1. 从头尾找到相同前缀/后缀（A、D不动）
2. 中间乱序部分：标记哪些旧节点在新列表中还存在
3. 计算最长递增子序列 → 这些节点不需要移动
4. 只移动不在 LIS 中的节点 → 最小化 DOM 移动
```

### Vue 3 其他优化

| 优化 | 说明 |
|------|------|
| **PatchFlag** | 编译时标记节点哪些部分会变化（文本、class、style 等），diff 时跳过静态部分 |
| **静态提升** | 静态节点提升到渲染函数外，不重复创建 |
| **Block Tree** | 只追踪动态节点，形成"扁平"的动态节点列表，跳过整棵静态子树 |

## 六、Vue 2 vs Vue 3 Diff 对比

| 特性 | Vue 2 | Vue 3 |
|------|-------|-------|
| 比较策略 | 双端比较 | 双端比较 + LIS |
| 静态优化 | 无 | PatchFlag / Block Tree |
| 编译时优化 | 无 | 有（静态提升、树扁平化） |
| 移动判断 | 逐个查找移动 | LIS 最小化移动次数 |
| 复杂度 | O(n) | O(n)，但常数更小 |
